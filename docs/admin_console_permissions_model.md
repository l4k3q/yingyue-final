# 后台控制台、权限菜单与模型引擎实现说明

## 实现范围

- 控制台：展示用户、角色、功能、模型、Token、采集记录、数据仓库和默认模型状态。
- 用户管理：支持新增、修改、删除、禁用、搜索和分页；`admin` 超级管理员受保护。
- 角色与权限：用户和角色保持一对一关系，角色通过功能树授权后台能力。
- 功能管理：功能启停后会影响角色授权和后台访问判断。
- 菜单管理：左侧菜单由当前登录用户的角色授权动态生成，只展开当前活动菜单；菜单管理页保留功能与菜单映射列表。
- 模型引擎：支持 OpenAI API 范式的生文、生图、生视频模型配置，支持默认模型、SSE 测试和 Token 统计。

## 权限规则

- `admin` 是超级管理员，初始化时同时写入 `admins` 和 `users` 表。
- 任何用户都不能删除、禁用或修改 `admin` 的用户名、角色和权限。
- 只有 `admin` 本人可以修改自己的密码。
- 非 `admin` 后台用户必须拥有至少一个启用的 `/admin/*` 功能授权才能登录后台。
- 后台路由会按角色授权拦截；例如 `用户管理员` 可以进入控制台和用户管理，但不能进入模型引擎。
- 被禁用的功能不会出现在授权树和动态菜单中，也不会通过路由权限校验。
- 角色权限配置页使用服务端渲染的树形复选框，避免前端组件初始化失败导致权限树空白。

## 默认数据

- 默认角色：
  - 普通用户：用户侧登录角色。
  - 系统管理员：后台全功能角色。
  - 用户管理员：后台用户管理角色。
- 默认账号：
  - `admin / admin888`
- 默认模型：
  - `DeepSeek Free`，类型为 `text`，可在模型引擎中替换真实 API Key 后测试。

## 模型引擎配置注意事项

- New API / OpenAI 兼容网关的 `Base URL` 通常需要填写到 API 前缀，例如 `https://你的网关域名/v1`。
- 如果只填写网关首页域名，生图测试可能拿到 New API 前端 HTML，而不是模型 JSON。
- 生图模型会调用 `/images/generations`，生视频模型会调用 `/videos/generations`。
- 生图返回 `data[].url` 或 `data[].b64_json` 时，测试面板会直接渲染图片。
